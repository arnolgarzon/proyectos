ARQUITECTURA CLIENTE/SERVIDOR

La arquitectura cliente - servidor es un modelo el cual se basa en dos actores: uno con el rol de proveedor de recursos y otro con el rol de consultor sobre los recursos.

Teoria

cliente

Programa ejecutable que se encarga del establecmiento de las conexiones. Envia una peticion al servidor y espera el regreso de una respuesta

Servidor

Es un programa que ofrece un servicio que se puede obtener en una red. El servidor comienza su ejecucion antes de empezar la interaccion con el ciente.

Bibliografia: http://profesores.elo.utfsm.cl/~agv/elo322/1s16/projects/reports/Proyecto%20Cliente%20-%20Servidor.pdf

Caracteristicas

cliente

  • Es quien inicia las solicitudes o peticiones
  • Espera y recibe las repuestas del servidor
  • Tiene la capacidad de conectarse a varios servidores a a vez
  • Interactua con el usuario final mediante una interfaz grafica

Servidor

  • Esperan a que lleguen las solicitudes de los clientes
  • Tras la recepcion de una solicitud, la procesan y luego envian la respuesta al cliente
  • Por lo general reciben un gran numero de conexiones

Ventajas

  • Centralizacion del control
  • Escalabilidad
  • Facil mantenimiento
  • Existen tecnologias, suficientemente desarrollados, que aseguran a seguridad de las transacciones
  • En las redes c/s los demas clientes no tienen acceso a las IP's por lo que se dificulta el rastreo y/o hackeo de los usuarios

Desventajas

  • Cuando una gran cantidad de clientes envian peticiones al mismo tiempo crean una congestion en el servidor
  • El paradigma de c/s clasico no tiene la robustez de una red P2P. Cuando un servidor esta caido, las peticiones de los clientes no pueden ser realizadas
  • El software y el hardware de un servidor son generalmente muy determinantes
  • El cliente no dispone de los recursos que puedan existir en el servidor
  • En las redes c/s la unica forma de obtener informacion es a traves de lo que proporciona el servidor por lo tanto los clientes no pueden compartir informacion entre ellos

Â¿Porque la arquitectura c-s es importante en la programacion web?

La arquitectura cliente/servidor es importante en la programacion web ya que es una arquitectura en la que se basan la mayoria de paginas web en el momento permitiendo subir el contenido a un servidor y estar disponible en cualquier parte del mundo por el cliente.

Que es el Front End

El Front End es la parte de una pagina web o aplicacion que realiza interaccion con el usuario. Basicamente es todo la parte grafica.

Que es el Back End

El Back End es la parte interior de las aplicaciones por eso se le denomina el lado del servidor. Basicamente es la programacion que esta detras de las paginas web o aplicaciones.
